CSS动画之——animation

发表于 2017-04-05 | | 阅读次数 | 次

CSS的animation可以从一个CSS样式转换到另一个。动画由两部分组成——用来描述CSS动画的样式和表示动画开始和终止状态的关键帧的集合。

相对于原始的脚本驱动的动画来说，CSS animations有以下三个优势：

  • 对于简单的动画非常适用：开发者不需要使用JavaScript就可以创建动画
  • 在中等的系统中加载也运行良好：使用JavaScript实现简单的动画可能会有较差的表现。渲染引擎能够通过frame-skipping和其他技术让动画尽可能流畅
  • 将动画序列交给浏览器控制，从而让浏览器对性能进行优化：例如，降低不可见部分动画的刷新频率

animation的配置

去创建一个CSS动画序列，你要通过animation属性或者它的子属性对需要应用动画的元素进行样式设定。你可以设置定时、持续时间和其他一些你想要在动画过程中展现出的细节。但是不能设置动画的实际展现形式，因为它们将在@keyframes中被定义。

animation属性的子属性

animation-delay

设置从元素加载完毕到动画开始之间的时间间隔

animation-direction

定义在一次动画序列运行结束后，动画是改变方向还是从起点开始重复

animation-duration

设置动画完成一次循环需要多长时间

animation-iteration-count

设置动画需要重复的次数；设置infinite可以无限循环

animation-name

指定@keyframes的名字

animation-play-state

可以暂停和恢复动画序列

animation-timing-function

设置动画的时间函数(加速曲线)

animation-fill-mode

设置在执行之前和之后动画要应用的值

使用 keyframes 定义动画序列

在设置了动画定时之后，你需要定义动画的表现。这通过使用@keyframes规则建立两个或者更多的关键帧来实现，关键帧使用百分比表示动画过程中的时刻。0%表示动画序列的第一个瞬间，100%就代表动画的终止状态。因为这两个状态太重要了，因此它们有特殊的别名：from和to。它们都是可选的。如果from/0%和to/100%没有指定，浏览器将会使用所有属性的计算值作为动画的开始和终止。

你也可以在动画的起始和终止之间有选择地添加一些中间关键帧。

使用animation事件

你还能够从另外一种方式控制动画，那就是animation事件。AnimationEvent对象能够被用来监听动画开始、结束以及新循环的开始。每个事件包括它发生的时间以及触发事件的动画的名称。
